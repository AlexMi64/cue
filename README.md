# cue

**Приватный ИИ-помощник поверх экрана для macOS и Windows.**

cue видит экран, распознаёт разговоры и помогает отвечать на вопросы, проводить встречи и решать задачи по программированию.

</div>

> [!IMPORTANT]
> cue предназначен для собственных заметок, обучения, доступности и тренировок. Использование скрытого помощника на прокторинге, собеседовании или записываемой встрече может нарушать правила платформы и требования к согласию на запись. Ответственность за использование программы лежит на пользователе.

## Возможности

cue работает как небольшая панель поверх других окон и использует три независимых источника данных:

| Возможность | Как запустить | Что используется |
|---|---|---|
| Помощь | `⌘` `↵` или `Ctrl` `Enter` | экран и недавний разговор |
| Что сказать? | кнопка в панели | звук встречи и микрофон |
| Уточняющие вопросы | кнопка в панели | весь разговор |
| Итоги | кнопка в панели | весь разговор |
| Задать вопрос | введите текст и нажмите `Enter` | экран и разговор |
| Решить задачу | `⌘` `H` или `Ctrl` `H` | экран |
| Умный режим | переключатель в поле ввода | более мощная модель |

Дополнительно в форке доступны русский интерфейс, русский язык ответов, копирование сообщений, трей, красный индикатор записи и Windows-сборка.

## Установка готовой версии

### macOS

1. Откройте раздел [Releases](../../releases) и скачайте архив `cue-mac.zip`.
2. Распакуйте архив и перенесите `cue.app` в папку «Программы».
3. При первом запуске нажмите по приложению правой кнопкой и выберите «Открыть».
4. Если macOS сообщает, что приложение повреждено, выполните:

```bash
xattr -cr /Applications/cue.app
```

### Windows

Windows-установщик собирается автоматически через GitHub Actions. Готовый `.exe` можно скачать во вкладке [Actions](https://github.com/AlexMi64/cue/actions) из артефакта `cue-windows`.

## Запуск из исходников

Требуется Node.js 18 или новее.

```bash
git clone https://github.com/AlexMi64/cue.git
cd cue
npm install
npm start
```

Сборка:

```bash
npm run pack       # локальная папка с приложением
npm run dist:win   # Windows-инсталлятор
```

macOS-сборка:

```bash
npm run dist
```

## Первичная настройка

1. Запустите cue и откройте настройки через кнопку `...`.
2. Выберите провайдера и добавьте свой API-ключ.
3. Для этого форка endpoint по умолчанию: `https://byesu.com/v1/`.
4. Для Gemini используйте доступную через endpoint модель, например `gemini-3-flash`.
5. Разрешите доступ к микрофону и записи экрана в настройках операционной системы.

Ключи хранятся только локально в файле `cue-data.json`. В репозиторий ключи не добавляются.

### Распознавание речи

Gemini распознаёт аудио через мультимодальный запрос к `chat/completions`. OpenAI-транскрибация требует доступа ключа к Whisper или другой STT-модели. Если такого доступа нет, используйте Gemini или отдельный сервис распознавания речи.

## Управление

- `⌘` `↵` на macOS или `Ctrl` `Enter` на Windows - помощь по экрану и разговору.
- `⌘` `H` или `Ctrl` `H` - решить задачу программирования на экране.
- Кнопка с квадратом - начать или остановить запись.
- Красный квадрат означает, что запись активна.
- Кнопка «Скрыть» сворачивает панель до верхней полосы.
- `⌘` `⇧` `T` или `Ctrl` `Shift` `T` - скрыть приложение в трей.
- Иконка cue в трее показывает окно, открывает настройки и завершает работу.
- Кнопка копирования переносит в буфер весь текущий диалог.
- `⌘` `⇧` `X` или `Ctrl` `Shift` `X` - выйти.

## Как это работает

cue построен на Electron. Основная логика выполняется локально, а запросы отправляются только выбранному пользователем ИИ-провайдеру.

- экран захватывается через Electron `desktopCapturer`;
- микрофон записывается через `getUserMedia`;
- звук встречи захватывается через `getDisplayMedia` с системным аудио;
- аудио преобразуется в PCM 16 kHz и отправляется на распознавание;
- ответ модели поступает в панель потоково.

Приложение не имеет собственного сервера, аккаунтов, телеметрии и аналитики.

## Устранение проблем

**Не работает распознавание речи.**

Проверьте API-ключ Gemini и endpoint `https://byesu.com/v1/`. Для OpenAI проверьте доступ к Whisper. Также убедитесь, что операционная система разрешила доступ к микрофону и записи экрана.

**Не появляется звук встречи.**

Разрешите запись экрана. В Windows дополнительно проверьте разрешения приложений на доступ к микрофону и системному аудио.

**cue видно в демонстрации экрана.**

Скрытие от захвата является best-effort и не гарантируется всеми программами. Не используйте эту функцию для обхода правил платформ.

## Конфиденциальность

- нет аккаунтов, собственного сервера и телеметрии;
- API-ключи хранятся только на компьютере пользователя;
- экран и аудио отправляются провайдеру только при запуске соответствующей функции;
- текущая расшифровка хранится в памяти приложения и не сохраняется cue на сервере.
